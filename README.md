# SIEM, Splunk & Elastic Labs

## Overview

This repository documents my hands-on learning in SIEM platforms, log analysis, and alert triage using tools such as Splunk and Elastic. These labs helped me understand how security analysts investigate alerts and analyse logs in real-world environments.

## Labs Covered

- Introduction to SIEM
- Splunk: The Basics
- Splunk: Data Manipulation
- Splunk: Exploring SPL
- Splunk: Dashboards and Reports
- Splunk: Setting up a SOC Lab
- Log Analysis with SIEM
- Alert Triage with Splunk
- Alert Triage with Elastic
- Elastic Stack: The Basics
- ItsyBitsy
- Fixit

## Skills Demonstrated

- SIEM fundamentals
- Log analysis
- Alert triage
- SPL (Splunk query basics)
- Data filtering and correlation
- Dashboard and report awareness
- Security investigation workflow
- Incident documentation

## Tools Used

- Splunk (concepts and SPL basics)
- Elastic Stack (ELK)
- Log analysis platforms

## Investigation Workflow

1. Review the alert
2. Search related logs
3. Filter relevant events
4. Identify suspicious activity
5. Correlate events
6. Determine severity and impact
7. Document findings

## What I Learned

These labs helped me:

- understand how SIEM tools are used in SOC environments
- analyse logs to identify suspicious behaviour
- use search queries to investigate events
- understand how alerts are triaged and prioritised
- interpret data in dashboards and reports

## What I Would Do in a Real SOC

If a security alert was triggered, I would:

- validate the alert
- search for related indicators in logs
- analyse user, system, and network activity
- collect indicators of compromise (IOCs)
- determine severity and impact
- escalate confirmed incidents
- document the investigation clearly

## Key Takeaways

SIEM platforms are critical for:

- monitoring security events
- detecting suspicious activity
- supporting incident response
- improving visibility across systems
